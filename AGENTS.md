# WeStream Production - Project Rules

- **Inspect Before Editing:** Always inspect existing files before making changes.
- **Explain Plan:** Explain plan before broad changes.
- **No Fake Facts:** Do not invent client facts, services, metrics, or testimonials.
- **Reference Strategy:** Primary reference is https://www.moshimoshi.in/. Study it as a complete digital system. Internal routes are secondary examples.
- **No Cloning:** Use the reference system for inspiration only; do not clone code, copy layouts, scrape, or steal visual identity.
- **Git Permission:** Never push, commit, or merge code without explicit permission from Yusuf.
- **Dependency Discipline:** Do not install dependencies without a clear, documented reason.
- **Focused Changes:** Do not rewrite unrelated files during a task.
- **Accessibility:** Preserve accessibility (focus states, reduced-motion, ARIA where needed).
- **Performance:** Preserve performance; defer heavy media.
- **Docs Sync:** Keep documentation synchronized with architectural reality.
- **Centralize Placeholders:** Keep placeholders centralized and clearly labeled.
- **Clear Explanations:** Explain important changes in simple language.
- **Security:** Never expose secrets; never commit `.env` or `.env.local` files.
- **Architecture Integrity:** Never silently change the architecture.
